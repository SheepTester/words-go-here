Autocorrect-resistent JavaScript keyboard

In unfortunate circumstances, I sometimes want to write JavaScript on my phone, but autocorrect can get in the way of typing code. More specifically, I like to swipe type on mobile, so it's easy and quick to type normal, capitalized sentences like "I'm going to piss on your bed."

However, camelCased JavaScript variable names like getBoundingClientRect require manually typing out and casing each letter, which is comparatively a pain. In addition, after typing a period (.), the phone keyboard autocapitalizes the next word, which isn't great for JavaScript, a case-sensitive language. Also, for some reason, swyping const keeps giving me "Cindy."

This tool lets me swype an English-like sentence, then it'll do its best to turn it into valid JavaScript code by merging adjacent words into camelCase names.
